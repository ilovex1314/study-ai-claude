# F4 — 灵活权重

**问题**：当前每天题目权重和必须为 100，由作者手工保证。题量/权重一变就要重算，容易出错。

**范围**：让权重表达更灵活，同时保持"满分 100"的不变量。两种可选模式（择一或并存）：

1. **相对权重归一化**：作者写相对权重（如 3/2/2），生成时归一化到 100，校验器据此核对。
2. **难度系数**：按难度标签（easy/medium/hard）映射默认权重，作者只标难度。

**实现**：主要在 `scripts/generate-content.mjs` 生成阶段做归一化，并在 `scripts/validate-content.mjs` 调整权重校验逻辑（归一化后仍须等于 100）。题型注册表与评分核心不受影响。

**验收**：归一化后每天总分仍为 100；现有"绝对权重"内容继续有效；校验器对不能整除/归一化后非 100 的情况报错。
