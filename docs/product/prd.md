# study-ai-claude 产品需求文档（PRD）

## 一句话

一个本地优先的 AI 工程学习工作台，把"读完课程"升级为"形成可验证能力"。

## 背景与问题

`../study-ai` 从 13 天迭代到 20 天后积累了：死代码、多事实源漂移、固定 4 道单选题、内容偏浅、架构图脆弱、"可验证产出"从未落地。本项目从第一天就以单一事实源、深内容契约、关系正确的图引擎和可验证产出闭环避开这些问题。

## 目标用户

想做出 AI 产品（而非只学 AI 名词）的中高级前后端工程师；每天约 60 分钟，20 天完成一条能力依赖路线。

## 能力路线（20 天）

Day01–04 建立模型、RAG、Agent、产品交付基础；Day05–09 比较平台并成熟知识系统；Day10–13 加入评估、上下文、整合与设计验证；Day14–20 为数据治理、安全、多模态、成本、SRE、发布治理与产品战略做生产化加固。每天有能力目标、可验证产出、概念、架构图、生产案例/反例、动手练习、加权测验、复盘与参考资料。

## 范围

### P0.5 + F5（本次，已交付）

- 单一事实源内容管道 + 深内容契约 + CI 校验门禁。
- 关系正确可测的图引擎；直观展示层（渐进式层级、案例对比）。
- 加权测验（满分 100，先 `single` 题型 + 注册表接缝）。
- 可验证产出闭环：练习证据 + 自评 Rubric + 能力档案导出。
- learner-state 适配器（本地优先，统一 key）。

### 后续（已分阶段，未实现）

- **P1**：F1 多选、F2 填空、F3 问答（自评）、F4 灵活权重。
- **P2**：AI 教练（基于能力档案的下一步推荐）。
- **P3**：团队能力视图、内容治理、商业化评估。

### 范围外

登录、云同步、后端、AI 自动评分、团队视图。

## 关键决策

- frontmatter 装**结构化数据**（非解析正文）：题目/权重/边无歧义可测。
- 题型用**判别联合 + 注册表**：加题型 = 加 grader + renderer + 内容，不动核心评分。
- 问答题（F3）无后端 → 设计为**自评题**（对照参考答案 + rubric），复用练习 rubric 机制。
- 生成物 `lessons.generated.ts` **不入库**，prebuild/CI 生成。
- 缺参考资料/权重不兜底，**报错**让校验暴露。

## 验收

- `pnpm validate:content && pnpm test && pnpm build` 全绿。
- 20 天内容均通过深度契约；每天题目权重和为 100。
- 能力档案能由测验 + 练习证据 + Rubric 聚合并导出 JSON。
